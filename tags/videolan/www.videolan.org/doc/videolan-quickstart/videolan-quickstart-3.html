3. Step 2 : Install vlms and unicast or broadcast a file to vlc(s)

Now that you have a working vlc on your client workstations, you can install a vlms (VideoLAN mini Server) on your future VideoLAN Server. This computer must be a Linux and can be a pretty modest machine.

3.1 Install vlms

In order to install vlms, download the file vlms-version.tar.gz from the vlms download page.

Then, install vlms : 

% tar xvzf vlms-version.tar.gz
% cd vlms-version
% make
% su
 [enter root password]
# make install

You can also do a make clean or make distclean as needed.

3.2 Unicast to a vlc

Run vlms

Download this streamable MPEG 2 PS file for your tests : ftp://ftp.videolan.org/pub/videolan/streams/presentation/presentation_short.vob.

You are going to send the stream to a client which has vlc installed. The IP address of the client is for example 10.0.0.2. You can replace-it by the DNS name.

% vlms -d 10.0.0.2 presentation_short.vob

If you want the server to stream the file continuously, do : 

% vlms -d 10.0.0.2 -l presentation_short.vob

To get the list of command line options : 

% vlms --help

Run vlc

Run vlc on the client machine : 

% vlc udp:

You can also use the interface : 

% vlc
and then click on "Net" and select "UDP".

The Windows version of vlc has difficulty reading from the network under Windows 95/98/ME. vlc works well under Windows 2000 and XP.

3.3 Broadcast to vlc(s)

You can send the stream to all the machines of a subnet. All the clients of the subnet will be able to view the stream. This is pretty dirty as the machines who don't want to view the stream will receive the stream.

You will use the same method that you use for unicast. Replace the IP address of the client 10.0.0.2 with the Broadcast IP address of the subnet, in the command line of vlms.
