VideoLAN - Tests de diffusion sur la Rez

Si tu n'as aucune idée de ce qu'est VideoLAN

Tu es probablement GPA et viens d'arriver sur la rez, il y a peu de chances pour que tu aies déjà eu l'occasion d'utiliser VideoLAN. En bref : VideoLAN est un ensemble de logiciels, développé principalement par des centraliens, qui permet de diffuser des chaînes vidéo sur un réseau informatique. En pratique, cela signifie que chaque étudiant, depuis sa chambre, peut regarder tout ce que est diffusé à ce moment par les serveurs de VideoLAN (DVDs, chaînes satellite numériques, ou encore des évenements sur le campus retransmis en direct).

Personne n'est parfait

VideoLAN est encore en développement et il ne faut donc pas compter sur des diffusions sans faille; des coupures, ralentissements ou autres problèmes sont à prévoir : cela ne fait que quelques mois que les tests de diffusions sur le réseau entier de la résidence ont commencé. Ce n'est pas non plus un "service" aux connectés; les membres de VideoLAN gèrent les diffusions dans la mesure du temps libre qu'ils ont à y consacrer.

Ceux chez qui ça ne marchera pas...

  • Tout d'abord, tu dois être connecté sur le nouveau réseau de VIA. A ce jour, le bâtiment J, le 0G, le 2G, le 3G et les assoces ne sont pas encore sur le nouveau réseau.
  • Ton ordinateur doit être cadencé à au moins 400 MHz et avoir 64 Mo de RAM minimum.
  • Les cartes réseau de VIA pour ordinateurs portables (les 3Com PCMCIA) ont du mal à supporter un flux réseau important et occupent alors beaucoup de ressources processeur. Sur une majorité de portables dotés de ces cartes, cela risque d'être saccadé. Si ton portable est puissant, ce ne sera probablement fluide que sur les flux dont le débit n'est pas trop élevé trop gros (comme les flux satellites, par exemple).

Nécessaire

Pour utiliser VideoLAN, il suffit d'installer un lecteur multimédia : le Client VideoLAN (vlc). Le vlc est capable de lire :

  • des fichiers MPEG 1, MPEG 2 et MPEG 4 / DivX,
  • des DVDs, sans aucune considération de zone géographique,
  • des vidéos depuis le réseau, ce que nous utiliserons ici.

Installer et lancer le Client VideoLAN

sous Windows

Assure-toi que ta version de DirectX est supérieur ou égale à 7. Si tu as une version trop vieille, installe la dernière version de directX disponible sur bipbip.

Télécharge l'installeur Windows et double-clique dessus pour lancer la procédure d'installation.

Pour le lancer, il suffit de double-cliquer sur le cône !

Attention, sous Windows 2000 / XP, il vous faut les droits d'administrateur sur votre système pour pouvoir lire depuis le réseau.

Sous Windows 95 / 98 / ME, le vlc est instable, et peut faire planter la machine. Nous te conseillons d'installer une version plus robuste de Windows, comme Windows 2000 ou XP Pro.

sous Mac OS X

Télécharge le package MAC OS X et double-clique sur l'icone du package pour lancer la procédure d'installation.

Pour le lancer, il suffit de double-cliquer sur le cône !

sous Linux Debian Woody ou Sid

Edite en root le fichier /etc/apt/sources.list et rajoute la ligne suivante :
deb ftp://ftp.videolan.org/pub/videolan/debian ./

Tape ensuite en root :
apt-get update
apt-get install libmad0 a52dec-dev libdvdcss2 libdvbpsi1 vlc vlc-gnome vlc-mad

Pour le lancer, il suffit de taper vlc.

sous Linux Mandrake, RedHat ou SuSE

Installe les packages a52dec and libmad0, qui devraient se trouver dans ta distribution. Télécharge le RPM de libdvdcss sur la page de téléchargement de libdvdcss et installe-le. Télécharge le RPM de lidvbpsi sur la page de téléchargement de libdvbpsi et installe-le. Télécharge les RPM vlc vlc-gnome et vlc-mad sur la page de téléchargement du vlc et installe-les.

Pour le lancer, il suffit de taper vlc.

sous Unix / Linux

Réfère-toi au VideoLAN Quickstart, section 2.1, paragraphe "Unix or Linux".

Comment se servir du client VideoLAN ?

sous Windows ou Linux

Une fois que le vlc est lançé...

Cliquer sur l'icône Net et remplir la boîte de dialogue comme ci-dessous :

Cliquer sur "OK". Une nouvelle section de l'interface apparaît :

Sélectionne un numéro de chaîne et clique sur "Go!".

Pour passer en plein écran, appuie sur "f". Pour revenir à une fenêtre video, appuie de nouveau sur "f".

Pour changer de langue ou de sous-titre, fait un clic droit sur l'image et sélectionne la langue ou le sous-titre à la souris.

sous Mac OS X

Tu ne peux malheureusement pas utiliser l'interface pour lire un flux réseau. Il faut lancer le vlc en ligne de commande et aller chercher les adresses IP multicast sur la page Web avec le programme. Par exemple, pour voir la chaîne 1, ouvre un terminal, mets-toi dans le répertoire où se trouve l'exécutable du vlc (normalement /Applications/vlc.app/Contents/MacOS/) et tape :

./vlc udp:@239.255.30.11

Pour passer d'une chaine d'une chaîne à l'autre, il faut quitter (appuier sur "q") et relancer le vlc avec une autre adresse IP multicast.

Qu'est-ce qui est diffusé et sur quelle chaîne ?

Une page listant les programmes diffusés est disponible à l'adresse http://vlcs/vls/index.phtml. Il y a même des liens hypertextes qui pointent vers les programmes des chaînes satellite.

Que faire si ca ne marche pas ?

  • Si rien ne s'affiche, c'est peut-être qu'il n'y a aucun flux dans cette chaîne... Essaye d'autres chaînes !

  • Vas dans le menu "View" puis "Messages..." :

    Regarde alors les messages d'erreur indiqués en rouge. Ici, il y a marqué :

    Network error: no answer from vlcs

    Cela signifie que le VideoLAN Channel Server est injoignable, ce qui correspond souvent à une coupure de courant au local VideoLAN. En haut de cette page, un message mis-à-jour automatiquement indique si le local VideoLAN a de l'électricité ou pas...

  • Le Client VideoLAN plante ! Si tu es sous Windows 95 / 98 / ME, c'est un problème connu. Nous te conseillons de passer à une version plus robuste de Windows comme 2000 ou XP pro. Sinon, regarde le paragraphe suivant...

  • Tu es sous Linux et ça ne marche pas ? Vérifie que tu as bien compilé le support du Multicast dans ton noyau. Cette option se trouve dans le menu "Networking Options" et s'appelle "IP: multicasting". Pour plus d'informations sur la compilation d'un noyau, lis la format ion Linux n°2.
