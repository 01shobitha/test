Compile VLC on BeOS

Needed tools:

  • You need BeOS 5 or higher with developpement tools.
  • Download and untar to /boot/home/config the archive of needed tools. It includes tools for the configure and build process (perl 5.8.0, autoconf 2.57, automake 1.7.2, libtool 1.4.2) and external libraries that the VideoLAN client uses (a52dec 0.7.4, ffmpeg's libavcodec 0.4.6, libdvdread 0.9.3, libfaad CVS-20030130, libogg 1.0, libvorbis 1.0, mad 0.14.2b, libdvdcss CVS-20030130 and libdvdplay CVS-20030130).
  • In order to use these libraries, add the following lines to your /boot/home/.profile:
    export CFLAGS=-I/boot/home/config/include
    export CPPFLAGS=-I/boot/home/config/include
    export CXXFLAGS=-I/boot/home/config/include
    export LDFLAGS=-L/boot/home/config/lib
  • Not necessary but convenient: gcc 2.95.3 and cvs.

Now you can compile vlc:

  • Download the source code (use cvs or download the archive)
  • cd to the source directory
  • run ./bootstrap
  • run ./configure. The following is suggested:
    ./configure --enable-a52 --enable-ffmpeg --enable-dvdplay --enable-faad --enable-mad --enable-ogg --enable-vorbis --enable-debug
  • run make
  • If you are using the old gcc 2.9 that comes with BeOS 5, the compilation may fail on a few modules. It is a gcc bug, you can avoid it by using gcc 2.95.3 or by decrease the optimization level from -O3 to -O0.
