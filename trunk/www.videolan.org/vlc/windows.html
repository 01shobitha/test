vlc on Windows

If you are using vlc on Windows, have a regular look at this page to see how the port is going along. vlc now work on Windows 95/98/ME and 2000/XP.

Current status

Program compiling: yes
Additional requirements: none
Plugin support: yes
CPU-specific acceleration: MMX, MMX EXT, SSE, 3D Now!
Native interface: Native, Gtk+
Native video output: DirectX, SDL
YUV acceleration in video output: yes
Native audio output: DirectX, WaveOut, SDL
File input: yes
Unencrypted DVD input: yes
Encrypted DVD input: yes
VCD input: no
MPEG-4/DivX support: yes
IPv4 Network input: yes
IPv6 Network input: yes
HTTP protocol : yes
Satellite card : no

Build process

The Windows binaries are cross-compiled under Linux using the mingw32 cross-compiler. If you want to compile the directx, gtk and sdl plugins you will also need to download additionnal development packages.
For complete instructions on how to build vlc for Windows, take a look at the Cross Compile HOWTO.

You can download all the necessary tools from here:
linux mingw32 cross-compiler. This version has been built from gcc-2.95.3-20011106, binutils-2.11.92-20011113, mingw-runtime-1.2 and w32api-1.2 found on mingw web site.
gtk development package for win32. This package has been built from glib-dev-1.3.12-20020101.zip, libiconv-dev-1.7.zip, libintl-0.10.40-20020101.zip and gtk+-dev-1.3.0-20020207.zip found on Tor Lillqvist web site (http://www.gimp.org/~tml/gimp/win32/new-downloads.html) .
SDL-devel-1.2.3-mingw32.tar.gz From www.libsdl.org.
The directx development package on dx7headers.tgz.

Known issues

We now have a pretty good Windows client. It's functional, quite fast, and it eats much less CPU than the Windows Media Player on a MPEG or DivX file.
