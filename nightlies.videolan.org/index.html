VLC media player nightly builds

Here you can download experimental versions of VLC that are daily generated from the VLC code that is currently in development. Several issues that existed in previous versions of VLC might be fixed here, but since this version of VLC is not yet finished, you should expect other problems and issues.

  • Windows builds

  • WindowsCE / PocketPC builds

  • Mac OS X PowerPC builds

  • Mac OS X Intel builds

  • BeOS builds

  • Debian Sid I386 (unstable): add the following line to your sources.list 
    deb http://nightlies.videolan.org/build/sid-i386/arch ./

    or download here

  • Debian Sid AMD64 (unstable): add the following line to your sources.list 
    deb http://nightlies.videolan.org/build/sid-amd64/arch ./

    or download here

    Those repositories contain a vlc-dbg package that you can install so you can run gdb to provide usefull backtrace when reporting bugs.

    Our debian archives are signed using this key (81CACA84). To use it, either :

    • Retrieve it directly (but you don't really check the key):
      As root, run : 
      wget -O /dev/stdout http://nightlies.videolan.org/key|apt-key add -
    • Import it in your gnupg installation, either from the file or from any keyserver, and check it (it is signed by several VideoLAN developers).
      	gpg --keyserver subkeys.pgp.net --recv-keys 81CACA84
	gpg --list-sigs 81CACA84 (check for errors)
	gpg --export 81CACA84 -a|sudo apt-key add -

  • Ubuntu Breezy I386 (stable) snapshot: add the following line to your sources.list 
    deb http://nightlies.videolan.org/build/breezy-i386 /

    or download here

    This package is not automatically built every night.

    Our breezy archive is signed using this key (81CACA84). To use it, you may follow the same steps as for our debian archive's key.

    or download here

    This package is not automatically built every night.

    Our dapper archive is signed using this key (81CACA84). To use it, you may follow the same steps as for our debian archive's key.
