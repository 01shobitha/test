    • VLC for Linux

    You have two choices:

    • download a set of packages for your architecture / distribution. You can choose the set of plugins that you'd like to use with vlc and download their individual packages. This option is the easiest one, but some exotic configurations may not have a binary package yet.
    • download a source tarball of vlc and compile it yourself. This will work in all cases, but you'll need additional packages and you should be very familiar with the build process of your architecture.

    See the VLC source code.

    Latest Debian x86 packages (0.5.0) are below. You may also add the following lines to your /etc/apt/sources.list:

    deb ftp://ftp.videolan.org/pub/videolan/debian $(ARCH)/
    deb-src ftp://ftp.videolan.org/pub/videolan/debian sources/

    Firstly, VLC is included in MandrakeLinux distribution, thus if your are running cooker, just get it from you preferred cooker mirror.

    As it is not possible to update softwares in a previous distrib, you have to install the rpms by "hand". The simplest way is using urpmi (command line interface) or rpmdrake (you will find in the Mandrake Control Center). General instrctutions about urpmi can be found here: http://www.urpmi.org (both in french and english). urpmi-setup will make easier to setup urpmi to your needs: http://plf.zarb.org/~nanardon/ . You need at least to add main, contrib and plf sources.

    • get libdvdplay, vlc, vlc-plugin-mad, vlc-plugin-a52, gnome-vlc (or vlc-gtk) rpm from VideoLAN site and all the plugins you want.
    • install them with: urpmi libdvdplay*rpm *vlc*rpm

    urpmi will complete all the dependencies.

    If you want to do all by hand, get all the required rpm from a mirror: http://www.mandrakelinux.com/fr/ftpurl.php3 but you should really use urpmi !
