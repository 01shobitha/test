Compile VLC on Mac OS X

Compiling VLC on Mac OS X is different from normal linux compiling and OS X compiling. We do not use Project Builder to compile, but a simple ./configure and then make isn't enough either. The instructions below are always for the current release.

If you compile from Subversion then please remember that Subversion can often be in an unstable state. If you wish to develop or test please join the vlc-devel mailinglist.
Don't put any of the non released test binaries or svn-compiles on software sites or on userforums. We have had bad experiences with this before and we do not appreciate it.

Please follow these steps for compiling:

  • Install de Mac OSX developer tools from the Developer Tools CD, or get them online.
  • Download the source code (use subversion)
  • cd to the source directory
  • Read the file README located in vlc/extras/contrib and choose whether to compile contrib from source or binary.
  • If you have fink, either disable it by commenting the line #source /sw/bin/init.csh or modify the /sw/bin/init.csh to add /sw/bin path AFTER $path
  • If you use Subversion, then run ./bootstrap
  • run ./configure --disable-x11 --disable-xvideo --disable-gtk --disable-sdl --enable-ffmpeg --with-ffmpeg-mp3lame --with-ffmpeg-faac --enable-mad --enable-libdvbpsi --enable-a52 --enable-dvdplay --disable-dvdnav --enable-faad --enable-vorbis --enable-ogg --enable-theora --enable-faac --enable-mkv --enable-freetype --disable-cddax --disable-vcdx --enable-speex --enable-flac --enable-goom --enable-livedotcom --enable-caca --disable-skins2 --enable-debug
  • make
